🌟 New Concept: Merge Sort

Today you're starting your first Divide and Conquer algorithm.

This is one of the most important sorting algorithms.

Companies like:

Google
Microsoft
Amazon
Adobe

love asking questions based on Merge Sort.

Why Do We Need Merge Sort?

Suppose we have:

arr = [8,3,6,1,5,2]

Bubble Sort

O(n²)

Selection Sort

O(n²)

Insertion Sort

O(n²)

Merge Sort

O(n log n)

This is much faster for large arrays.

Core Idea

Merge Sort follows:

Divide

Split the array into two halves.

[8,3,6,1,5,2]

↓

[8,3,6]

[1,5,2]

Split again.

[8]

[3,6]

[1]

[5,2]

Split until every array has only one element.

[8]
[3]
[6]
[1]
[5]
[2]
Conquer

Now merge them back in sorted order.

[3]
[6]

↓

[3,6]
[5]
[2]

↓

[2,5]

Eventually:

[1,2,3,5,6,8]
Visualization
            [8,3,6,1]

          /           \

      [8,3]         [6,1]

     /    \         /    \

   [8]   [3]     [6]    [1]

     \    /        \     /

     [3,8]        [1,6]

         \         /

       [1,3,6,8]

Notice something?

This is our first recursive algorithm that splits a problem into smaller problems.

Important Observation

Merge Sort has two separate jobs:

1. Split the array (Recursion)
2. Merge two sorted arrays

You already learned how to merge two sorted arrays a few days ago.

So half of Merge Sort is already familiar to you.

Today's Goal

We're not implementing Merge Sort today.

We're first mastering the merge step.

This will make the recursion much easier tomorrow.

⭐ Q5 (Thinking Question)

Without writing code, answer:

When one array finishes first, what should happen to the remaining elements of the other array?

For example:

arr1 = [1,4]
arr2 = [2,3,5,6]

After merging 1,2,3,4, what should happen to 5 and 6?
